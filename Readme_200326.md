## 1. Fix Avatar trên Sơ đồ Tổ chức

### Vấn đề
- Avatar upload lên disk nhưng HTTP 404 vì NestJS chưa bật serve file tĩnh.
- Frontend dùng relative path `/uploads/avatars/xxx.jpg` → browser gọi sai port (3000 thay vì 3001).

### Giải pháp
**`apps/api/src/main.ts`** — Bật Express static middleware:
```ts
const express = app.getHttpAdapter().getInstance();
express.use('/uploads', require('express').static(path.join(process.cwd(), 'uploads')));
```

**`apps/web/src/components/org-chart/custom-nodes/structure-node.tsx`** — Thêm `resolveAvatarUrl()`:
```ts
function resolveAvatarUrl(avatar?: string | null): string | null {
    if (!avatar) return null;
    if (avatar.startsWith('http')) return avatar;
    return `${getApiOrigin()}${avatar}`; // prefix API origin tự động
}
```
- Dùng `API_BASE_URL` có sẵn → tự động đúng mọi môi trường (dev/staging/prod).
- Fallback sang `ui-avatars.com` nếu ảnh thật không tải được.

## 2. Fix Lightbox Avatar (React Portal)

### Vấn đề
- Lightbox render bên trong ReactFlow node → mouse events bubble lên canvas → canvas pan/dịch chuyển.
- Nút X không đóng được, di chuột thì lightbox "chạy loạn".

### Nguyên nhân kỹ thuật
```
Mouse move → bubble → ReactFlow Canvas → PAN gesture → canvas dịch chuyển
```

### Giải pháp: `createPortal`
```tsx
// Mount lightbox trực tiếp vào document.body — ngoài ReactFlow hoàn toàn
{typeof document !== 'undefined' && createPortal(lightboxContent, document.body)}
```

**Các biện pháp bổ sung:**
- `onPointerDown`, `onMouseDown`, `onMouseMove`, `onWheel` → `stopPropagation` trên backdrop
- `useEffect` → Escape key đóng lightbox + lock body scroll
- Avatar lightbox: **180px** (tăng từ 112px), card width: **400px** (tăng từ 340px)
- Hiển thị đầy đủ: Ảnh → Tên → Chức vụ → Email (icon xanh) → SĐT (icon xanh lá)

## 3. Fix Employee Panel — Sai số lượng nhân sự theo đơn vị

### Vấn đề
- `selectedNode.id` là `undefined` vì node data từ API không có trường `id`.
- Kết quả: employee panel không filter được nhân sự theo unit đúng.

### Giải pháp
**`apps/api/src/modules/employee/services/employee-query.service.ts`** — Thêm `id: item.id` vào `buildNodeData`:
```ts
const buildNodeData = (item: any, type: string) => ({
    id: item.id,   // raw UUID — dùng để filter nhân sự ở frontend
    label: item.name,
    ...
});
```
- Xóa cache `org_chart_structure` ngay để load data mới (không chờ 10 phút expire).

## 4. Mở rộng Dropdown Bộ phận trong Form Nhân viên (Solution 2)

### Vấn đề
- Factory/Division/Department/Section dropdown chỉ hiện đơn vị cùng `companyId` → thiếu lựa chọn khi nhân viên làm việc tại đơn vị khác công ty.
- Ví dụ: Nhân viên Mekong không thấy Nhà máy Nhật Việt; Chỉ thấy KHỐI SẢN XUẤT không thấy các khối khác.
- TypeScript lỗi: `Property 'companyId' does not exist on type 'Department'` và `Section`.

### Giải pháp: Bỏ filter `companyId` — hiện toàn bộ
**`apps/web/src/components/employees/employee-form.tsx`**

| Dropdown | Trước | Sau |
|---|---|---|
| **Nhà máy** | Filter theo `companyId` | Tất cả + `[Tên Công ty]` trong label |
| **Khối** | Filter theo `companyId` / `factoryId` | Tất cả |
| **Phòng ban** | Filter theo `companyId` | Tất cả; cascade lọc theo Khối nếu đã chọn |
| **Bộ phận** | Filter theo `companyId` | Tất cả; cascade lọc theo Phòng nếu đã chọn |

**Đánh giá rủi ro:**
- ✅ Zero DB changes, zero data migration
- ✅ Ảnh hưởng đúng 1 component (employee-form.tsx)
- ✅ Không ảnh hưởng Quản lý suất ăn, Quản lý thiết bị, IT Ticket
- ✅ Chênh lệch hiệu suất < 5ms (tất cả factories/divisions < 50 bản ghi)
