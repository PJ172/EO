## 2. BACKEND — Cập nhật Service (6 file)

### 2.1 `apps/api/src/modules/documents/documents.service.ts`
- Thêm `where: { deletedAt: null }` vào `findAll()` và `findOne()`
- Thêm method `delete(id, userId?)` — xóa mềm

### 2.2 `apps/api/src/modules/projects/projects.service.ts`
- Lọc `deletedAt: null` trong `findAll()` và `findOne()`
- `remove()` → xóa mềm **cascade**: xóa dự án → tự động xóa mềm toàn bộ công việc (cùng `batchId`)
- `removeTask()` → xóa mềm công việc đơn lẻ

### 2.3 `apps/api/src/modules/news/news.service.ts`
- Lọc `deletedAt: null` trong `findAllArticles()`, `findPublishedArticles()`, `findOneArticle()`
- `deleteArticle()` → xóa mềm thay vì xóa cứng

### 2.4 `apps/api/src/modules/it-asset/it-asset.service.ts`
- Lọc `deletedAt: null` trong `findAll()`
- `delete()` → xóa mềm, thêm hậu tố `_DELETED_{timestamp}` vào mã tài sản để tránh trùng

### 2.5 `apps/api/src/modules/ticket/ticket.service.ts`
- Lọc `deletedAt: null` trong `findAll()` và `getStatistics()`
- Thêm method `deleteTicket(id, userId?)` — xóa mềm
- Cập nhật SLA compliance query để loại bỏ ticket đã xóa

### 2.6 `apps/api/src/modules/roles/roles.service.ts`
- Lọc `deletedAt: null` trong `findAllRoles()`
- `deleteRole()` → xóa mềm (giữ nguyên permissions để có thể khôi phục đầy đủ quyền)

## 3. TRASH MODULE — Viết lại hoàn toàn

**File:** `apps/api/src/modules/trash/trash.service.ts`

Mở rộng từ **5 module → 11 module**. Cập nhật toàn bộ 7 chức năng:

| Chức năng | Mô tả |
|-----------|-------|
| `getTrashItems(module)` | Liệt kê các mục đã xóa theo module (phân trang) |
| `getTrashSummary()` | Đếm số lượng mục đã xóa ở tất cả 11 module |
| `restoreItem(module, id)` | Khôi phục 1 mục (kiểm tra trùng mã trước khi khôi phục) |
| `restoreBatch(batchId)` | Khôi phục nhóm mục bị xóa cùng lúc (cascade restore) |
| `hardDeleteItem(module, id)` | Xóa vĩnh viễn 1 mục |
| `emptyTrash(module?)` | Dọn sạch thùng rác theo module hoặc toàn bộ |
| `cleanupExpired(days)` | CronJob tự dọn dẹp dữ liệu quá hạn (mặc định 30 ngày) |

### Các module được hỗ trợ:
`employees`, `departments`, `users`, `jobTitles`, `factories`, `documents`, `projects`, `newsArticles`, `itAssets`, `tickets`, `roles`

## 5. LOGIC ĐẶC BIỆT

| Logic | Chi tiết |
|-------|---------|
| **Cascade Delete** | Xóa dự án → xóa mềm tất cả task con (cùng `batchId`) |
| **Cascade Restore** | Khôi phục dự án → khôi phục task con bị xóa cùng batch |
| **Chống trùng mã** | Thêm `_DELETED_{timestamp}` khi xóa, gỡ khi khôi phục, kiểm tra conflict |
| **Bảo toàn quyền** | Xóa mềm vai trò giữ nguyên `RolePermission` → khôi phục không mất quyền |
| **Tự dọn dẹp** | CronJob xóa vĩnh viễn mục trong rác > 30 ngày |
