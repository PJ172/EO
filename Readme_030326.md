# Nhật ký cập nhật ngày 03/03/2026

## 1. Khắc phục sự cố Throttling Timeout (Code lỗi: "Too Many Requests")
- **Nguyên nhân chính:** Ứng dụng Web cũ khi thực hiện xóa hàng loạt (Bulk Delete) trên Grid thường dùng vòng lặp gửi hàng trăm API request đến Backend qua `axios` hoặc `Promise.allSettled`. Việc này gây kích hoạt limit rate chặn chống spam của NestJS (`@nestjs/throttler`).
- **Giải pháp:**
  - Nâng giới hạn config Throttle trong quá trình khởi tạo ứng dụng: `short` (10 request/giây -> 50), `medium` (100 request/giây -> 500) trên phạm vi global (`apps/api/src/app.module.ts`).
  - Lập trình Endpoint Backend mới dành cho các module chưa hỗ trợ Bulk Delete (VD: `POST /users/bulk-delete` trong `apps/api/src/modules/users/users.controller.ts`).
  - Gắn code nghiệp vụ sử dụng `Prisma.$transaction` ở lớp Service tương ứng giúp hệ thống xử lý gộp chung transaction DB, an toàn và tối ưu tài nguyên (`users.service.ts`).
  - Cập nhật lại logic hàm `handleBulkDelete` ở giao diện `apps/web/src/app/(dashboard)/settings/users/page.tsx`, sử dụng API endpoint Bulk Delete mới giúp xóa 500+ hàng trong tích tắc. 

## 2. Giải quyết tình trạng Đóng Băng Giao Diện (UI Lag) khi Chọn Nhiều Dòng
- **Nguyên nhân chính:** Thành phần `TableRow` ở trạng thái nguyên thủy là một state component. Nếu người dùng check "Select All" hoặc thay đổi mảng `selectedRows`, React sẽ kiểm tra lại từng thành phần trong `TableBody` rồi so khớp virtual DOM khiến Web client bị đứng vài giây với mảng > 500 records.
- **Giải pháp:**  
  - Tạo một Higher Order Component và bọc bằng thư viện `React.memo` cho các dòng Table (`MemoizedUserRow` trong `users/page.tsx` và `MemoizedOrgRow` trong `org-module-page.tsx`).
  - Viết custom-comparator để React hiểu rằng chỉ nên thay đổi Virtual DOM của Row nếu Checkbox trạng thái `isSelected` của chính ID dòng thay đổi chứ không render hàng loạt.
  - Fix hiện tượng ngậm ứng dụng trên tất cả table toàn cục.

## 3. Quản trị giao diện Dialog hiển thị sai lệch tâm
- **Sự cố:** Mặc định hộp thoại mở lên lần đầu bị lệch về góc trái trên màn hình, thay vì hiện ở giữa. Rất rõ ràng khi thao tác tính năng Import/Upload Excel.
- **Giải quyết:** Vào tận root level Component Dialog UI (`apps/web/src/components/ui/dialog.tsx`) cấu hình fallback fallback sizes và bọc cơ chế tính toán trong `requestAnimationFrame` đối với component `<DialogContent />`. React hiện tại sẽ được cấp thêm thời gian để vẽ hoàn tất `boundingBox` nhằm lấy toạ độ chính xác 100% giúp popup hoàn toàn trung tâm trên mọi màn hình.
