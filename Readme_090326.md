## Chi tiết các thay đổi

### 1. Fix lỗi Infinite Render Loop (React)
- **Vấn đề:** Màn hình Quản lý Phòng ban (`/departments`) bị lỗi màn hình đỏ `Maximum update depth exceeded`.
- **Nguyên nhân:** Mảng `departments` tự động được khởi tạo mới `[]` trên mỗi lần render, khiến hook `useEffect` phụ thuộc vào nó bị kích hoạt liên tục không ngừng.
- **Giải pháp:** Sử dụng `useMemo` để ghi nhớ tham chiếu của mảng `departments` thay vì gán giá trị fallback trực tiếp (`const departments = useMemo(() => departmentsData?.data || [], [departmentsData?.data]);`).
- Tối ưu hóa điều kiện cập nhật state trong `useEffect` để tránh re-render thừa. Cùng lúc đó, rà soát lại tất cả các trang khác (như Employee, Company) và xác nhận các trang đó không bị lỗi tương tự.

### 2. Sửa lỗi Giao diện Hiển thị Số lượng Nhân viên (Bất đồng bộ)
- **Vấn đề:** Màn hình Quản lý Phòng ban hiển thị số lượng nhân sự rất lớn (ví dụ 240 người) trong khi Màn hình Nhân viên trống trơn (0 người).
- **Nguyên nhân:** Do dev-server cũ vẫn giữ cached Prisma Client bản cũ không có schema `company`, `division`, `section`, nên API `/employees` trả về lỗi 500 (bị sập), trong khi thuật toán đếm dồn gộp (`_count.employees`) trên API `/departments` gọi hàm count của SQL nên vẫn hoạt động bình thường.
- **Giải pháp:** Tắt ứng dụng Node đang treo cổng 3001 và khởi động lại dev server để nhận cấu trúc Prisma mới nhất. Đồng thời chạy test trên toàn bộ API (bằng `npx tsc`) để đảm bảo quá trình tái cấu trúc CSDL không làm chết các query của các modules cũ (như Report, Dashboard, KPI). Tỉ lệ tương thích đạt 100%.

### 3. Nâng cấp File Mẫu Nhập Liệu (Excel Template) - Phase 2
- Gỡ khoá (unprotect) toàn bộ các ô nhập liệu trong file Excel để người dùng không gặp lỗi khi Paste dữ liệu.
- Cập nhật Data Validation dạng Combobox (Dropdown) trên file Excel Nhập/Xuất Nhân viên cho mọi cấp Tổ chức: Công ty, Nhà máy, Khối, Phòng ban, Bộ phận, Chức vụ.
- Thiết kế Data Dictionary linh động dạng `Mã Tổ Chức | Tên Tổ Chức` (ví dụ `CT001 | Thái Dương`) để người dùng dễ nhận biết tổ chức theo tên thay vì phải nhập Code khô khan.
- Bổ sung tùy chọn mặc định `Không có` ở trên cùng mỗi Combobox.
- **Tính năng Đột phá:** Tắt bảng thông báo lỗi (`showErrorMessage: false`) trong Data Validation của ExcelJS, kết hợp với các trình Excel đời mới (Office 365) giúp biến Combobox thả xuống thành **Danh sách Lọc (Autocomplete Search)**. Người dùng có thể đánh chữ vào ô Excel và danh sách sẽ tự lọc các kết quả tương ứng rất thân thiện với 1 Database Tổ chức khổng lồ.
- Nâng cấp code Import trên Backend (`employee.excel.service.ts`) để tự động bẻ chuỗi string (Bằng ký tự `|`) nhằm lọc lấy ngược `Code` xử lý vào hệ thống nội bộ từ format UI thân thiện.

### 4. Sửa lỗi Xung đột Cổng và Biến Tham chiếu Database (PostgreSQL)
- **Vấn đề:** Ứng dụng thi thoảng báo sai mật khẩu Admin (401 Unauthorized) hoặc import dữ liệu Nhà Máy (Factory) báo thành công nhưng lại hiển thị 0 dòng.
- **Nguyên nhân:** Khác biệt Case-Sensitivity (viết hoa/thường) trong các file `.env` nội bộ của dự án (`eoffice_dev` vs `eOffice_Dev`). Điều này vô tình tạo ra **hai Database độc lập** chạy song song. Tiến trình API NestJS lưu cache kết nối tới Database cũ kẹt ở cổng 3001.
- **Giải pháp:** 
  - Đồng bộ hoá toàn bộ file `.env` (của NextJS và NestJS) về chung một tên schema chuẩn là `eoffice_dev`.
  - Hủy tiến trình chạy ngầm (Zombie Process) trên cổng 3001 và khởi động lại luồng `npm run dev` để ép Prisma Client nạp thông số máy chủ mới.
  - Cập nhật lệnh xoá dữ liệu nhanh `clear-db.ts` chuyển sang dùng `TRUNCATE CASCADE` dọn hoàn toàn các bảng liên quan đến Cơ cấu tổ chức mà không vi phạm khoá ngoại (Foreign Key constraint) trong khi vẫn giữ lại dữ liệu bảng `users`, `roles`, `permissions`.
  - Reset mật khẩu tài khoản quản trị viên (`admin`) về `Admin@123` trực tiếp qua script mã hoá của `bcrypt` nhắm thẳng tới trường `passwordHash`.
