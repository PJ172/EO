# Nhật ký cập nhật EOffice (Ngày 25/03/2026)

## 1. Đồng bộ Giao diện (UI/UX)
- Cập nhật Component Header (`PageHeader`) của các phân hệ tuân thủ tuyệt đối quy tắc **Purple Ban**, phân định dải gradient màu riêng biệt cho từng phân hệ (VD: News dùng màu đỏ, ISO dùng màu xanh lá...).
- Đồng bộ nút chức năng trên Data Table: Các phân hệ (Cars, News, ISO Documents, Thống kê, Roles...) đã được gộp các nút "Xuất/Nhập Excel" chuyên biệt vào cụm Menu Dropdown "Tùy chọn".
- Xóa lỗi hiển thị và chuẩn hoá lại font hiển thị viết hoa ở phần Tiêu đề.

## 2. Hoàn thiện tính năng Import/Export (Excel Templates)
- Đổi tên cấu hình trên DB từ `PJ - Import` / `PJ - Export` thành `Import` / `Export` trên toàn bộ hệ thống.
- Chỉnh sửa kết nối API, thay đổi logic query tìm kiếm (Search) cấu hình sang dạng không phân biệt hoa/thường (`insensitive`).
- Cải thiện UX File Mẫu Excel: 
  - Thêm tính năng **Combobox tự động thả xuống** (Data Validation) để chọn dữ liệu tham chiếu như: Công ty, Nhà máy, Khối, Phòng ban, Bộ phận, Chức danh, Vị trí công việc thay vì nhập liệu thủ công.
  - Các phần tử bị đánh dấu "Ẩn khỏi bộ lọc" (`excludeFromFilters = true`) sẽ tự động không hiển thị trong Excel.
  - Chuyển `Sheet Danh Mục` - tài nguyên cho Combobox - ở trạng thái `veryHidden` để tránh bị tương tác nhầm.

## 3. Quản lý Vị trí công việc (Job Positions)
- Khởi tạo Data Data Table chuẩn cho quản lý Vị trí công việc (Job Positions).
- Bổ sung cấu hình hiển thị Vị trí công việc lên Sơ đồ tổ chức qua biến cờ `showOnOrgChart`. Hỗ trợ chức năng cập nhật hàng loạt trên Data Table.
- Khắc phục lỗi `filter is not a function` với trang Job Positions thông qua việc đồng bộ cấu trúc Object trả về từ Axios Paginated.

## 4. Sơ đồ tổ chức (Org Chart)
- **Hiển thị Vị trí công việc**: Sơ đồ hiện tại đã hiển thị nội dung trường "Vị trí công việc" ngay phía bên dưới Chức danh, tương quan đến dữ liệu của Backend API.
- Cập nhật cơ sở hạ tầng Prisma Scheme cho phép Mapping và kết nối trường dữ liệu `currentPosition` với mỗi biểu mẫu (EmployeeNode).
- Khắc phục các vấn đề liên quan đến TypeScript typing của Prisma Client.

## 5. Dọn dẹp & Tái cấu trúc
- Di chuyển các trang giao diện về đúng nhóm phân quyền: `settings/`, `general/`, `hr/`, `ga/`, `iso/`, `cntt/`.
- Xoá bỏ các Route không còn sử dụng theo lộ trình Re-Structure (Leaves, KPI, timekeeping, projects, ...).
