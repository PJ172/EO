## II. Sửa lỗi phân quyền xóa (Delete Permission)

### Vấn đề
- User `it` có quyền Administrator nhưng không xóa được dữ liệu
- Nguyên nhân: Permission guard kiểm tra `user.roles.includes('ADMIN')` — phân biệt hoa thường với JWT payload

### Giải pháp
- Đảm bảo permission guard bypass đúng với role code `'ADMIN'` từ JWT payload
- Administrator bypass tất cả permission check → có thể xóa, restore, force-delete mọi record
- File: `apps/api/src/modules/auth/guards/permissions.guard.ts`

## IV. Sửa lỗi TypeScript (TS1272)

### Vấn đề
9 lỗi `TS1272` trong 3 controller mới:
```
error TS1272: A type referenced in a decorated signature must be imported
with 'import type' or a namespace import when 'isolatedModules' and
'emitDecoratorMetadata' are enabled.
```

### Nguyên nhân
NestJS với `isolatedModules: true` + `emitDecoratorMetadata: true` yêu cầu **interface** dùng trong decorator (`@Body()`, `@Query()`) phải import bằng `import type`.

### Giải pháp
Tách import trong cả 3 controller:
```typescript
// Trước (lỗi):
import { CompanyService, CreateCompanyDto, UpdateCompanyDto, CompanyQueryDto } from './company.service';

// Sau (đúng):
import { CompanyService } from './company.service';
import type { CreateCompanyDto, UpdateCompanyDto, CompanyQueryDto } from './company.service';
```
